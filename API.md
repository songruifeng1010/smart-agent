# SmartAgent API文档

## 概述
SmartAgent是一个智能对话助手，提供了以下API端点，用于处理用户的对话请求和管理会话。

## 基础URL
默认情况下，API端点的基础URL为：`http://localhost:3002`

## API端点

### 1. 对话API

#### 端点：`POST /api/chat`

**功能**：处理用户的对话请求，返回智能助手的响应。

**请求体**：
```json
{
  "message": "你好，你是谁？",
  "sessionId": "optional_session_id"
}
```

**参数**：
- `message` (必填): 用户的消息内容，字符串类型。
- `sessionId` (可选): 会话ID，字符串类型。如果提供，将使用该会话ID管理对话上下文；如果不提供，将使用默认会话。

**响应**：
```json
{
  "response": "我是SmartAgent，一个智能对话助手。"
}
```

**示例**：
```bash
curl -X POST http://localhost:3002/api/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "你好，你是谁？"}'
```

### 2. 健康检查API

#### 端点：`GET /health`

**功能**：检查服务器的健康状态。

**响应**：
```json
{
  "status": "ok",
  "timestamp": "2026-04-13T12:00:00.000Z",
  "uptime": 3600
}
```

**示例**：
```bash
curl http://localhost:3002/health
```

## 会话管理

SmartAgent支持会话管理，通过会话ID可以保持对话的上下文。会话默认超时时间为1小时，超过1小时未使用的会话将被自动清理。

### 会话ID格式
会话ID必须满足以下条件：
- 长度在10-100个字符之间
- 只能包含字母、数字、下划线和连字符

### 会话操作

#### 创建会话
当首次使用一个会话ID时，系统会自动创建该会话。

#### 获取会话
可以通过调用`GET /api/session/{sessionId}`获取会话信息（此端点暂未实现，将在未来版本中添加）。

#### 删除会话
可以通过调用`DELETE /api/session/{sessionId}`删除会话（此端点暂未实现，将在未来版本中添加）。

## 错误处理

API返回的错误响应格式如下：

```json
{
  "error": "错误消息"
}
```

常见的错误码：
- 400: 请求参数错误
- 403: 禁止访问
- 404: 资源不存在
- 413: 请求体过大
- 500: 服务器内部错误

## 安全注意事项

1. 不要在请求中包含敏感信息，如API密钥、密码等。
2. 会话ID应保持私密，避免被他人获取。
3. 不要发送包含恶意代码的消息，系统会自动过滤不安全的内容。

## 速率限制

目前API没有速率限制，但为了系统稳定性，建议不要频繁发送请求。
